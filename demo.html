Option 5: Constant border-radius + Corner Fill Overlays

Problem: Dynamically changing border-radius triggers Chrome layout recalculation, which thrashes with 100+ DOM elements.

Solution: Keep border-radius: 24px on the content panel at all times. To make a corner appear square, position a 24×24 overlay (pseudo-element on the wrapper) in that corner, filled with the panel's background color. Toggle the overlay's opacity (0→1) to show/hide it. Since border-radius never changes and opacity is compositor-only, zero layout is triggered.
